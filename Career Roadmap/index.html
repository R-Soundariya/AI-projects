2. The 90 days

Twelve weeks, roughly 15 hrs/week. Each month ends with a shipped project — that's deliberate, not a stretch goal.

MONTH 01

Foundations — SQL, stats, Python

Weeks 1–2
SQL foundations
SQLBolt (free, interactive) → Mode Analytics SQL Tutorial (free) → daily reps on HackerRank SQL / StrataScratch free tier. 45–60 min/day.
Weeks 3–4
Statistics intuition + Python basics
StatQuest (YouTube) for intuition, Khan Academy Statistics & Probability for structure, Kaggle Learn: Python micro-course for hands-on reps.
📦 Capstone: Project 1 — Ops/support ticket analysis (see §3)
MONTH 02

Depth — pandas, visualization

Weeks 5–6
Pandas deep dive
Kaggle Learn: Pandas + Data Cleaning micro-courses; Corey Schafer's pandas playlist (YouTube) as a standing reference.
Weeks 7–8
Visualization / Power BI
Microsoft Learn: Data Analytics with Power BI (free path); Guy in a Cube (YouTube) for the practical technique the docs skip.
📦 Capstone: Project 2 — Retail sales dashboard (see §3)
MONTH 03

Differentiation — AI crossover, resume, applying

Weeks 9–10
AI-analyst crossover + resume v1
DeepLearning.AI: Prompt Engineering for Developers (free) to formalize what you already do; build Project 3; draft resume + LinkedIn with STAR-method stories from your ops role.
Weeks 11–12
Interview prep + start applying
SQL scenario drills (StrataScratch / LeetCode free tier), 2–3 self-recorded mock case interviews. Start applying from week 10 — don't wait for week 12 to feel "ready."

3. Portfolio projects

Three projects, each proving a different claim: domain fluency, technical range, and the AI-analyst edge almost no other applicant will have.

1

Ops/support ticket analysis

Analyze a support-ticket dataset (Kaggle's "Customer Support Ticket Dataset," or data shaped like what you already see at work) in SQL to find SLA-breach patterns and resolution-time bottlenecks.

SQLExcel

"Analyzed N months of support ticket data (SQL) to identify SLA-breach patterns; recommended a change projected to cut average resolution time by X%."

2

Retail sales dashboard

Clean a public retail dataset (Kaggle "Online Retail" or "Superstore") in pandas, then build an interactive Power BI dashboard with a short written case study alongside it.

PythonpandasPower BI

"Built an end-to-end sales analysis pipeline (Python/pandas, Power BI) surfacing seasonal demand and high-return categories; proposed changes to reduce returns by X%."

3

"Chat with your data" AI assistant

A small app — same stack as your grocery-list and trip-planner projects — that lets someone ask plain-English questions about a CSV and get pandas-generated answers and charts back, via the Claude API.

Claude APIPythonpandas

"Built an LLM-powered data assistant (Claude API + Python/pandas) answering natural-language questions over a dataset — applied experience in the AI+analytics overlap now showing up in analyst job descriptions."
